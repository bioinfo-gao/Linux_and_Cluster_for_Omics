源码包(开源的文件都是源码包)
  含脚本安装包(install.sh)
二进制包(计算机识别和运行的只有二进制包, 源码包经过编译形成二进制包)
  RPM包(CentOS中的二进制包) : redhad package mangement
  deb包(Debian中的二进制包)
  系统默认包(因为光盘离的默认Linux 按照包就是二进制包)

二进制包不再能看到源代码
但是现场编译很慢，二进制包不需要编译，安装迅速

二进制包是厂商或者机构或者软件作者已经进行了编译

目的是为了加快安装速度，简化安装过程

源码包的优点：
开源，顶级工程师的工作，可以学习和修改
开业自由选择功能
编译安装，更加适合自己的系统，更加稳定效率也更高
生成环境中，5%的效率提示就很可观，很重要
卸载方便(直接删除源码包即可，不会有任何垃圾遗留)

源码包的缺点：
安装步骤过多，尤其是大型软件集合，容易出错(尤其是拼写错误)
编译慢（一个软件20min 常见， Gentoo 安装需要3天）
编译安装的报错对新手很难解决

RPM包

二进制包的优点
包管理简单， 只要几个命令就可以安装升级查询
安装快速

二进制包的缺点
已经编译，不再能看到源代码
功能选择不如源码包灵活
依赖性
