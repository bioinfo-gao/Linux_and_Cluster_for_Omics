   1.1 命令全小写
   1.2 一些环境变量全大写
2. 目录名和文件名不允许有空格
3. 路径名中不能有空格
4. 路径名中不能有非法字符，如：/、\、:、*、?、"、<、>、|等
5. 路径名中不能有中文字符
6. 路径名中不能有连续的多个斜杠（/）或反斜杠（\）
7. 一切皆文件,包括硬件 , windows 是靠硬件管理器管理硬件, linux 是靠文件系统管理硬件
8. 一切修改,包括系统配置,只有写入配置文件才能永久生效, 否则重启系统则会丢失所有用命令修改的配置
9.  命令行中输入命令时，命令名和参数之间用空格分隔，命令名不区分大小写，但是参数区分大小写

14. 古老windows 有 8.3规则(3个字符的扩展名, 现在已经突破), 但linux 并没有限制文件名长度, 扩展名也不是必须,  在 Linux/UNIX 中：文件扩展名更多的是一种**“建议”或“惯例”**，而不是强制要求。
    
15. 在 Windows 中：文件扩展名是至关重要的。它在很大程度上决定了文件的类型和行为。如果你把一个 .exe 文件的扩展名去掉，你就不能直接双击运行它了。

16. 所有存储设备挂载之后才能， 包括硬盘、U盘、光驱等。 包括自动挂载和手动挂载。

17. Linux 系统中，所有文件都以块设备的形式存在，即使是普通文件也不例外。

18. Linux 系统中，所有文件都有权限属性，可以控制文件对其他用户的访问权限。

19. Linux 系统中，所有文件都有属主和属组，可以控制文件归属和共享。

20. Linux 系统中，所有文件都有时间戳属性，可以记录文件的创建、修改、访问时间。

21. Linux 系统中，所有文件都有链接属性，可以创建指向同一文件的多个别名。

22. Linux 系统中，所有文件都有大小属性，可以记录文件大小。

23. Linux 系统中，所有文件都有其他属性，可以记录文件类型、链接数、设备号等信息。

24. Linux 系统中，所有文件都有路径属性，可以记录文件在文件系统中的位置。

25. Linux 系统中，所有文件都有名称属性，可以记录文件名。

26. Linux 系统中，所有文件都有元数据属性，可以记录文件属性。

27. Linux 系统中，所有文件都有内容属性，可以记录文件内容。

28. windows 下的程序不能直接在 linux 下运行，需要先转换为 linux 格式的可执行文件。
